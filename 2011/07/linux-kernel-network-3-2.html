Linux Kernel 功能详解 网络篇 3

网络篇 3 Transformation user configuration interface

Networking support—>Networking options—>Transformation user configuration interface

Transformation user configuration interface 在 2.6.13 之前的名字是 IPsec user configuration interface，知道这个就很好理解了。这个选项支持用户层面的 IPsec 工具对内核进行调试。如果不选择这个，racoon ， openswan（老版本）应该是无法工作的。

在很多文章中都有指出，Openswan 对 Kernel 的配置需求如下：

Transformation user configuration interface

  • PF_KEY sockets

  • IP: AH transformations

  • IP: ESP transformations

  • IP: IPComp transformations

  • IP: IPsec tunnel mode

    • Networking support—>Networking options—>Transformation sub policy support

    关于这个选项可以找到的资料很少，除了 Kernel 的 Help，仅有一份 Mail。

    根据这些资料可以或者：

  • 1.此选项主要给开发人员使用。

  • 2.开启子策略支持。

  • 3.主策略与子策略同时对一个数据流生效。

  • 4.主策略与子策略在Kernel内生效时间上有所差别（这句的含义不太明确）。

    • Networking support—>Networking options—>Transformation migrate database
    同样，资料很少。
    我个人的理解是支持了对于动态节点的实时更新。在 Help 中所据的列子是，在 IPV6 环境下，Mobile 节点可能会不断的变化自己的上网地点（attachment point）。这时候此选项会提供保持 IPSEC 链接的能力。

    Networking support—>Networking options—>Transformation statistics
    此功能给出了所有 IPSEC 中错误包的统计数据，是给开发者使用的。
