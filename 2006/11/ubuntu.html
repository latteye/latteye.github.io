Ubuntu

初用Ubuntu，感觉非常不错。虽然自带的软件包不多，但在支持、人性化等方面都做的非常好。我努力在这里记录一下我所知道的关于Ubuntu的一切技术资料。方便自己查找。Ubuntu有很好的社区支持，他的中文社区在http://forum.ubuntu.org.cn/，他的Wiki在http://wiki.ubuntu.org.cn/。

一、光盘下载

有DVD和CD提供下载。

CD版本在安装后期需要连接网络来解决一些语言问题。DVD版本不需要。但比较而言Ubuntu光盘所提供的软件包数量不多。

二、安装

安装系统实在Live 的环境下进行的，这在显示以及语言等方面都带来了极大的方便。整个安装过程除了挂载硬盘个人感觉不够体贴，其他都非常好。

四、字体修改
直接使用Windows字体，美化系统。

1.将Windows目录下simsun.ttc tahoma.ttf verdana.ttf verdanab.ttf verdanaz.ttf tahomabd.ttf verdanai.ttf字体复制到/usr/share/fonts/XXX/下。XXX为文件名，可以自己任意建立一个。

mkfontscale mkfontdir fc-cache

五、安装XGL

XGL主页在http://www.compiz.org/，以下将罗列Ubuntu下的安装方法。

1.修改/etc/apt/sources.list文件。插入以下内容：

`deb http://www.beerorkid.com/compiz dapper main
deb http://media.blutkind.org/xgl/ dapper main
deb http://ubuntu.compiz.net/ dapper main

deb http://www.beerorkid.com/compiz edgy main-edgy
deb http://media.blutkind.org/xgl/ edgy main-edgy
deb http://ubuntu.compiz.net/ edgy main-edgy`

2.导入Key

wget http://www.beerorkid.com/compiz/quinn.key.asc -O - &#124; sudo apt-key add - wget http://media.blutkind.org/xgl/quinn.key.asc -O - &#124; sudo apt-key add - wget http://ubuntu.compiz.net/quinn.key.asc -O - &#124; sudo apt-key add -

3.更新你的源

4.apt-get install compiz xserver-xgl libgl1-mesa xserver-xorg libglitz-glx1 compiz-gnome cgwd cgwd-theme

5.修改/usr/bin/startxgl.sh：

`
[servers]
Xgl -fullscreen :1 -ac -accel glx:pbuffer -accel xv:pbuffer & sleep 2 && DISPLAY=:1

Start GNOME

exec gnome-session`

6.chmod 755 /usr/bin/startxgl.sh

7.编写文件/usr/bin/startcompiz

`#!/bin/sh
killall gnome-window-decorator
wait

cgwd &
compiz —replace gconf miniwin decoration transset wobbly fade minimize cube rotate zoom scale move resize place switcher trailfocus water &`

chmod 755 /usr/bin/startcompiz

8.gedit /usr/share/xsessions/xgl.desktop

&#91;Desktop Entry&#93; Encoding=UTF-8 Name=XGl Exec=/usr/bin/startxgl.sh Icon= Type=Application

9.确保所有软件都为最新版本。

sudo nvidia-xconfig
10.关闭所有程序,按下 Ctrl-Alt-Backspace 以重启 X server. 如果您在按下Ctrl-Alt-Backspace后看见 NVIDIA 标志的屏幕闪过, 则说明您的驱动已经安装正确了.

3.RealPlayer
安装很简单，直接用apt。但是需要解决一个播放rmvb卡的问题。


1、在／usr/local/RealPlayer/share/locale/zh_CN／下有一个简短的README，其中提到

* 本版本中不包含 Alsa 和 esound 驱动程序。
+ 可以使用 AUDIO 环境变量设置用于播放的OSS 设备。
+ 例如 export AUDIO=/dev/dsp2”（原文）……

RealPlayer需要AUDIO环境变量去连接正确的声音设备，设置环境变量expert AUDIO=/dev/dsp1，再启动RealPlayer。(这个方案对于双声卡的用户比较有用)

2、因为内核默认的oss驱动对某些声卡支持的不好，而RealPlayer又默认使用oss 所以我们使用alsa模拟的oss就好了 即用aoss realplay启动RealPlayer
为了方便 以后不用命令行 打开K菜单编辑器 在realplayer10那项的细节中把命令改称aoss realplay 保存
然后在控制中心的文件关联里 把rm/rmvb文件的默认打开方式全删掉 新浏览加入刚才改的菜单项 保存

以上内容转自http://www.cnitblog.com/lywaml/archive/2005/06/14/402.aspx

好了在ubuntu上这么操作。
apt-get install alsa-oss
接着附件-菜单编辑器-修改realplayer前面加上aoss就可以。完整命令为aoss realplay。

七、虚拟机

1.qemu

http://www.qemu.org 可以下载到最新的版本，包括kqemu。

首先需要安装编译环境：
sudo apt-get build-dep qemu

接着要安装内核的header。

编译两个软件包
qemu：
./configure --target-list=i386-user,i386-softmmu --enable-alsa --cc=gcc-3.4 --host-cc=gcc-3.4
kqemu：
./configure &amp;&amp; make &amp;&amp; make install

写一个 kqemu 加载工具 ~/bin/dokqemu.sh
`

!/bin/sh -v

mknod /dev/kqemu c 250 0
chmod 666 /dev/kqemu

/sbin/modprobe kqemu

lsmod | grep qemu
使用 qemu 前执行下sudo dokqemu.sh`

qemu 加参数 -kernel-kqemu
