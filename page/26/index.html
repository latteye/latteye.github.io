我的罗曼蒂克迷路史

今天早上又起晚了，在去公司的路上突然想到自己迷路的经验实在是太多了…… 写下来消遣消遣吧……

一、小屁孩篇

我的迷路史应该从小学开始说起，再早的我也不记得了。

好像我走路的步伐比一些朋友都要快，所以陈曦小朋友（小学、高中同学）会说：“你总是喜欢走在前面，一直都没变过。”冤枉啊！是你们走的太慢了…………

久而久之我和陈曦走在一起的时候就好像是我在带路，他跟着我走…… 事实上很多时候也的确如此。但问题出在！我不认路！

一天，放学。在距离学校不远的地方有一片老式的房屋。四层楼高度的小新村（那个时候还没有新村的叫法）。我们回家一直是路过那里，从来没有试着穿越过这个小新村。

我说：“走，我们今天从这里面走吧！”
陈曦：“你认识路么？”
我说：“认识！”（这一段有争议，我现在记得我没有说“认识”，但陈曦回忆的时候一口咬定是我说“认识”了，他才会跟我走的。）

于是两个小屁孩走入了从未探索的世界！

（唉……如果当时是个MM的话，我就会说 ‘从此王子和公主过着幸福快乐的生活’ ，可惜啊……）

通过网络安装 Linux （ CentOS ）系统

上班还是蛮忙的，所以最近LinuxBlog少有更新。
LinuxBlog 原则上不登 Server 性质的文章。但基于本文应用广泛，且耗费了我一天的时间，还是记录一下吧。

一、思路

无光驱、软驱的一台服务器，想要安装Linux系统。我们需要通过网卡的PXE协议，引导之后安装Linux。流程：机器启动－网卡引导－通过DHCP获得IP地址－通过tftp获得最基础的内核文件，使用该内核文件启动机器－启动之后可以对安装程序配置，选择使用http、ftp、nfs方式远程获得安装所需要的软件包。

显然，网络安装是必须配置服务器端的。我们的服务端需要提供以下服务：

DHCP
TFTP
HTTP(FTP,NFS)

一周小节

上班的第一周，我觉得有必要写个小节。新的环境会导致出现新的想法，或许以后会有更新的想法来否定目前的想法，但记录下来也会便于自己思考。

1.接触了新的环境，熟悉大环境比做具体工作更加重要。如果对大环境都不熟悉，在做具体工作的时候会遇到很多障碍。

2.《论语》－子曰：“不患人之不己知，患不知人也。” 我还做的不够好，太急躁了……

3.企业化的管理需要比较全面的思考，没有100%的把握不要随便动服务器。

4.Linux 的确不适合做桌面……尽管我之前很努力的想让桌面环境下普及 Linux，但我现在意识到 桌面 不止是 PC 这么简单。任重而道远……

公司里面需要集中式的管理，Windows 域可以管理几百万台计算机，并且较好的部署他们（相对）。Linux 可以么？如果真的桌面都用 Linux 管理员会累死的。

5.学习是不断的，新环境会出现无数新的知识，只有不断学习才能适应环境。除了复习AD等相关知识，我现想学习部署SMS。

6.需要一个好的方式来记录每天想到的问题，然后努力去解决他们。目前指发现 Google 文档 可以实现这样的工作过程。（个人需要）

7.处理好人际关系会给工作带来极大的便利。

呼呼～ 目前想到这么多，明天开始骑自行车上班！锻炼身体咯～
