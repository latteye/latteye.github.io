建立最简单的X环境

目前linux服务器上，一般比较少使用X环境， 但在某些情况下，又必须启用X才能完成一些任务，比如安装oracle。这种情况下当然也可以使用xhost将远程程序输出到本地X环境来显示。但我们今天的话题是如何快速安装X环境。
现在的X环境， 用的比较多的是gnome，KDE，还有相对轻量级的xfce。 但其实对真正服务器应用来说， X始终只是一个配角。
那如何部署最简单的X环境呢？其实以上提到的X环境（gnome，KDE，xfce）都是xinit进程加上窗口管理器和一些应用程序的组合而已。换言之，最简单的X就是xinit而已。

目前常见的linux系统，包括redhat，centos，debian，arch等，只需要安装xinit（当然还有它所依赖的包）即可。通过apt、yum都可以很容易的完成安装。从完全不带X的console加装X，只需20-30M就可以完成X环境部署。

Cisco 3560G 端口镜像

要设置 NTOP 监视公司 Wan 网流量，所以将 Fortigate — Cisco 端口机箱到 NTOP服务器—Cisco 端口。
同时将 NTOP 服务器网卡设置为混杂模式，这样就可以监视流量了。

Cisco configure 模式下：

monitor session 1 source interface 端口名 #此为Fortigate-Cisco端口

monitor session 1 destination interface 端口名 #此为NTOP—Cisco端口

另外看到一篇文章，介绍了许多种设置方法。不同型号的 Cisco 设置可能不同。不知道原理上有啥区别。

sendmail配置使用smtp服务器发邮件

公司里使用qmail和exchange来做邮件服务器，qmail做桥头，exchange做为后台服务器。
现在配置了一台linux作为bug track system，需要发送邮件给用户。系统默认使用的是sendmail，因为已经有了对外的qmail服务器，因此希望sendmail直接使用qmail作为smtp服务器发送邮件出去。

配置
sendmail
修改/etc/mail/sendmail.cf
将DS修改为
DS yourmailserver.com #yourmailserver.com为qmail服务器的fqdn名
然后重启sendmail

qmail
将sendmail所在服务器ip加入tcp.stmp文件中，然后qmailctl cdb就ok了
记得检查目的用户域是否已加入rcpthost文件
还有qmail默认会检查sender端的用户域，需要将配置好sendmail服务器发送域。因为我是内网服务，因此简单将nosendercheck设为1
