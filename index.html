Unity Content Size Fitter

作用

Content Size Fitter 的作用是调整该层布局本身的尺寸。而尺寸是由该 Game Object 上的布局组件的最小值或者最合适的值来调整。

请记住，当一个 Rect Transform 改变了大小，无论是通过 Content Size Fitter 还是别的控件，大小的改变都是基于中心点的。这意味着可以通过中心点来发出改变大小的指令。

比如，当中心点在中间时，Content Size Fitter 将全方位的扩展 Rect Transform，而当中新建在左上方的角落时，Content Size Fitter 将对 Rect Transform 向下和向右扩展。

属性

属性 作用
Horizontal Fit 如何控制宽度
None 不对布局宽度进行任何控制
Minimum 根据布局对宽度进行最小化的调整
Preferred 根据布局对宽度进行最优化的调整
Vertical Fit 如何控制高度
None 不对布局高度进行任何控制
Minimum 根据布局对高度进行最小化的调整
Preferred 根据布局对高度进行最优化的调整

Swift 中 的 as! 操作符

在 Swift 1.2 版本之前，as 操作符被用在两种不同的类型转换场合，取决于不同的转换源与目标，他会有不同的行为。

  • 保障性的转换，从一种类型转换为另一种类型时，Swift 编译器认为是安全的，可以提供保障的。比如大小写转换，数字转字符，1 转换为 浮点数等等。
  • 强制转换，从一种类型转换为另一种类型，Swift 编译器认为无法保障其安全的。比如从一个父类转换为子类等等。

在 Swift 1.2 版本，将这两种不同的转换行为使用两种符号来表示。as 表示这是一个安全的转换行为，而 as! 则表示这是一个强制的转换行为。当你看到!就表示这是有可能失败的转换行为，并有可能导致程序崩溃。
