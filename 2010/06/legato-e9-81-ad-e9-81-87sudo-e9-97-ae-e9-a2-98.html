Legato 遭遇 sudo 问题

2010年6月23日

今天一台legato备份出现一些问题，需要重启下服务。
因为同样的情况出现过好几回了， 每回重启后就ok了。
这次也就按部就班的执行

sudo /etc/init.d/gst stop sudo /etc/init.d/networker stop sudo /etc/init.d/networker start sudo /etc/init.d/gst start
执行的很顺利，可等了好一会，legato服务的监听还是没起来，查看日志，发现：
23 *** gstd starting at %s 1 36 24 Wed Jun 23 15:26:50 2010 2 %s 1 0 46 Internal error: could not get database handle. 2 %s 1 0 54 The most recently called subfunction returned error 2. 2 %s 1 0 50 Internal error: could not release database handle. 2 %s 1 0 27 library error 22 os error 0 23 *** gstd stopping at %s 1 36 24 Wed Jun 23 15:29:16 2010
打了技术支持的电话也没有什么办法
google下，发现有提到sudo引起的问题，依稀记得之前安装时使用sudo也出现过问题，于是切换到root下重新启动legato服务，果然立马解决。
