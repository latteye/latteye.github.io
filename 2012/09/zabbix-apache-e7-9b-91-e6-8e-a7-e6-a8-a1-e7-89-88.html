zabbix apache 监控模版

2012年9月4日

最近在调试 zabbix，去 wiki 转了一圈，在 Apache Monitoring Script 看到几个实现方式，我选择了方式二。就像作者说的，这种方式只需要部署在一点即可，不用每一个 agent 都有脚本。

跑了一下发现脚本已经无法运行，遂做了一点 debug，适当的修改了脚本和模版，在这里贴一下。我的环境是 CentOS 6 ，Python 2.6.6，zabbix 2.0. Apache 的状态页面路径为 /server-status 。将脚本放到 server 或者 proxy 的 external scripts 目录后，对应机器添加模版即可。这个脚本还有可以优化的地方是，端口应该作为一个变量来处理。
